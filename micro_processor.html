What is DMA?

Direct Memory Access lets an I/O device transfer data straight to RAM without going through the CPU. The CPU sets it up, then goes back to work while the DMA controller handles the heavy lifting.

Why DMA?

Without DMA, the CPU must read each byte from I/O and write it to RAM individually — thousands of instructions. DMA offloads this bulk transfer so the CPU can execute other programs concurrently.

Bus Grant (BG / BR)

BR (Bus Request): DMA asks CPU for control of the system bus.
BG (Bus Grant): CPU says "OK, it's yours" and tri-states (disconnects) from the bus.

Interrupt Signal

When all bytes have been transferred, the DMA controller fires an interrupt to the CPU. The CPU pauses its current task, checks which device interrupted, and handles the completion event.
